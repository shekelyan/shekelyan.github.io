Question: How does a system use the stack for function calls?

A Function calls do something akin to a context switch and store old contexts (current address, register values etc) in memory using the stack segment. B System does not have a stack, it's a data structure that could be used by some processes.
C Function calls jump to the code of the function and then (to not disturb prior register values) only use the stack segment in memory
